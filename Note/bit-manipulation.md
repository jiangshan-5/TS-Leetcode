# 🧩 位运算 (Bit Manipulation) 核心笔记

位运算是算法中极其高效且优雅的技巧。它直接操作计算机底层的二进制位，不仅速度快，而且往往能实现 $O(1)$ 的空间复杂度。

## 1. 基础运算符速查表

| 运算符 | 名称 | 规则 | 常用技巧 |
| :--- | :--- | :--- | :--- |
| `&` | 按位与 (AND) | 两个都是 1 才是 1 | `x & 1`: 判断奇偶; `x & -x`: 提取最低位的 1 |
| `\|` | 按位或 (OR) | 只要有一个是 1 就是 1 | `res \| (1 << i)`: 设置第 i 位为 1 |
| `^` | 异或 (XOR) | **不同为 1，相同为 0** | `a ^ a = 0`: 抵消成对数字; `a ^ 0 = a` |
| `~` | 按位取反 | 0 变 1，1 变 0 | `~x`: 配合求补码 |
| `<<` | 左移 | 所有位左移，右侧补 0 | `1 << i`: 制造第 i 位的掩码 |
| `>>` | 右移 | 所有位右移，左侧补符号位 | `num >> i`: 查看第 i 位的情况 |
| `>>>` | 无符号右移 | **所有位右移，左侧补 0** | `(a + b) >>> 1`: 计算中间值，防溢出且自动取整 |

## 2. 异或 (XOR) 的神奇魔法

异或是位运算题目中的绝对主角，主要利用其“**成对抵消**”的性质。

### 核心性质
1. **归零性**：`a ^ a = 0`
2. **恒等性**：`a ^ 0 = a`
3. **交换律/结合律**：运算顺序不影响结果。

### 经典应用：只出现一次的数字系列
- **找落单的一个数**：全员异或，成对的全部抵消，剩下的就是结果。
- **找落单的两个数**：
    1. 全员异或得到 `xorSum = a ^ b`。
    2. 通过 `lsb = xorSum & -xorSum` 找到 `a` 和 `b` 的第一个分歧位。
    3. 根据该分歧位将原数组分为两组，分别异或即可。

## 3. 按位统计法 (Bit-by-Bit Counting)

当数字出现的次数不是 2 次（例如 3 次、5 次）时，异或就不再适用。此时需要用到“按位统计”。

### 核心思想
- 整数由 32 个位组成。
- 统计所有数字在每一位（0~31）上 `1` 出现的次数。
- 如果某一位上 `1` 的总数不能被 `K`（重复次数）整除，说明答案在这一位一定是 `1`。

### 代码模版
```typescript
let res = 0;
for (let i = 0; i < 32; i++) {
    let count = 0;
    for (const num of nums) {
        if (((num >> i) & 1) !== 0) count++;
    }
    if (count % K !== 0) {
        res |= (1 << i);
    }
}
```

## 4. 进阶技巧与常用公式

### 判断是否为 2 的幂 (Power of Two)
*   **核心逻辑**：如果一个数是 2 的幂，它的二进制表示中**只有一个 1**。
*   **公式**：`x > 0 && (x & (x - 1)) === 0`
*   **原理**：`x - 1` 会将 `x` 最低位的 1 变成 0，并将其右边的所有位变成 1。二者相与，结果必然为 0。

### 汉明重量 (Counting Bits / Brian Kernighan's Algorithm)
*   **场景**：计算二进制中 1 的个数。
*   **技巧**：`n &= (n - 1)`
*   **原理**：每次执行此操作，都会消除 `n` 二进制表示中最右侧的一个 `1`。执行了几次，就有几个 `1`。

### 无符号右移计算中间值
*   **场景**：在二分查找或归并排序中计算 `mid`。
*   **技巧**：`const mid = (left + right) >>> 1;`
*   **优势**：
    1.  **防溢出**：即使 `left + right` 超过了 $2^{31}-1$，`>>>` 也会将其视为无符号正数处理。
    2.  **自动取整**：省去了 `Math.floor()`，性能更佳。

## 5. 位图集合 (Bitmask Set) 去重与状态判定

当我们需要判定一组状态，或者在**有限的集合内去重**时，如果集合的大小在 32 以内（例如 26 个小写英文字母），我们就可以使用一个普通的 `number` 类型（32位整型）来代替哈希表 `Set`，这被称为**位图状态机 (Bitmask Set)**。

### 📌 操作对照表

| `Set` 常规操作 | 位运算等价公式 | 含义 |
| :--- | :--- | :--- |
| `let set = new Set()` | `let mark = 0;` | 初始化为空集合 (所有二进制位为 0) |
| `set.has(x)` | `(mark & (1 << x)) !== 0` | 检查元素 `x` 是否已存在于集合中 |
| `set.add(x)` | `mark |= (1 << x);` | 将元素 `x` 放入集合中 (将第 `x` 位标记为 1) |

### 🛠️ 经典代码模板 (面试题 01.01 判定字符是否唯一)

```typescript
function isUnique(astr: string): boolean {
    let mark = 0; // 用一个 32 位整型作为布尔数组/哈希表
    for (const char of astr) {
        // 计算字符距离 'a' 的相对偏移量 (0~25)
        const moveBit = char.charCodeAt(0) - 97; // 'a'.charCodeAt(0) === 97
        
        // 1. 检查对应的 moveBit 位是否已经是 1 (是否重复)
        if ((mark & (1 << moveBit)) !== 0) {
            return false;
        }
        // 2. 将对应的 moveBit 位标记为 1
        mark |= (1 << moveBit);
    }
    return true;
}
```

### 🌟 优势
- **极小空间**：仅用 1 个 32位整型（4 字节）代替了一个哈希表对象，空间开销从 $O(N)$ 降为 $O(1)$。
- **极致速度**：位运算直接由 CPU 硬件指令支持，所有查/改操作均在 1 个 CPU 周期内完成，效率远超哈希表的哈希计算和冲突处理。
