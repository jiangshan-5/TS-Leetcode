# 🔗 链表算法核心摘要

> [!TIP]
> **可视化学习**：建议先查看 [🎨 链表算法可视化架构图](../Linked_List/linked_list_tutorial.excalidraw)，结合图示理解以下逻辑。

## 1. 物理本质
- **结构**：数据 (`val`) + 指向下一位的地址 (`next`)。
- **直觉**：它不是柜子，它是**寻宝游戏**。你必须拿到当前的线索，才能知道下一个地点。

---

## 2. 三大核心战术

### ① 哨兵节点 (Dummy Node) —— 逻辑降维
- **适用场景**：删除节点、合并链表。
- **奥妙**：强行给头节点安排一个“前任”，让“删除头节点”不再成为特殊情况。
- **金句**：只要涉及到“删除”，无脑先上 Dummy。

### ② 快慢指针 (Fast & Slow Pointers) —— 速度差
- **适用场景**：环检测、找中点、找倒数第 K 个。
- **奥妙**：利用两个指针的速度差或路程差，在不计算长度的情况下定位特定位置。
- **直觉**：操场跑步，快的终会追上慢的。

### ③ 路径对等 (Commutative Path) —— 数学浪漫
- **适用场景**：160. 相交链表。
- **奥妙**：`a + c + b = b + c + a`。
- **金句**：只要愿意去走对方走过的路，终究会相遇。

---

## 3. 开发避坑指南
1.  **存下家**：在修改 `curr.next = prev` 之前，必须先 `const nextTemp = curr.next`。
2.  **防空指针**：永远检查 `curr !== null` 和 `curr.next !== null`。
3.  **不丢头**：除非是 Dummy 节点，否则不要轻易移动 `head`，用 `cur` 去跑腿。
