# 🧠 动态规划 (Dynamic Programming) 学习笔记

## 1. 什么是动态规划？
动态规划（DP）是“**记住过去，推导未来**”的算法。它将复杂问题拆分为重叠的子问题，通过存储子问题的解（“填表”）来避免重复计算。

## 2. DP 核心五步法 (手写代码逻辑)
1.  **确定 dp 数组及下标含义**：
    - 例如：`dp[i]` 是爬到第 `i` 阶的方法数。
2.  **确定递推公式（状态转移方程）**：
    - 规律：`dp[i] = dp[i-1] + dp[i-2]`。
3.  **dp 数组初始化**：
    - 确定基础边界值，如 `dp[1] = 1`, `dp[2] = 2`。
4.  **确定遍历顺序**：
    - 通常是自底向上（从 0 到 n 的循环）。
5.  **举例推导验证**：
    - 模拟 `i=1, 2, 3` 的情况，看结果是否符合预期。

## 3. 开发避坑指南（TS/JS 实战经验）
- **【报错核心】二维数组初始化**：
  在操作 `dp[i][j]` 前，必须先确保 `dp[i]` 已经被初始化为数组。
  ```typescript
  for (let i = 2; i <= rowIndex; i++) {
      dp[i] = []; // 核心：先初始化当前行，再进行内部赋值
      for (let j = 0; j <= i; j++) {
          dp[i][j] = ...
      }
  }
  ```
- **越界问题**：内层循环的 `j` 范围要严格对应当前行的长度（通常是 `j <= i`）。

## 5. 经典递推与多指针协作模型 (以丑数 II 为例)

当我们要生成的序列中的每一个元素都**直接依赖于已生成序列中的某些子状态**，且依赖的倍数不同时，可以使用**多指针动态规划**。

### 📌 状态机设计与转移
以「丑数 II」为例，新的丑数必然由已有丑数乘 `2`、`3`、`5` 得到。我们维护三个指针：
* **定义**：`dp[i]` 为第 `i` 个丑数。
* **转移方程**：
  $$dp[i] = \min(dp[p_2] \times 2, \; dp[p_3] \times 3, \; dp[p_5] \times 5)$$
* **去重逻辑**：
  在将最小值 `minNum` 写入 `dp[i]` 后，使用多个并列的 `if` 分支对相应的指针递增。如果有多个指针算出的备选值相同（例如 `dp[3]*2` 和 `dp[2]*3` 都等于 `6`），这些指针会**同时向前移动**，自然完成了去重。

```typescript
function nthUglyNumber(n: number): number {
    const dp = new Int32Array(n + 1);
    dp[1] = 1;
    let p2 = 1, p3 = 1, p5 = 1; // 三个状态指针
    for (let i = 2; i <= n; i++) {
        const num2 = dp[p2] * 2;
        const num3 = dp[p3] * 3;
        const num5 = dp[p5] * 5;
        const minNum = Math.min(num2, num3, num5);
        dp[i] = minNum;
        // 并列 if 分支：更新可能产生相同值的多个指针，实现天然去重
        if (minNum === num2) p2++;
        if (minNum === num3) p3++;
        if (minNum === num5) p5++;
    }
    return dp[n];
}
```
