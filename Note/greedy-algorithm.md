# 贪心算法 (Greedy Algorithm) 学习笔记

## 1. 核心定义
贪心算法是指在对问题求解时，**总是做出在当前看来是最好的选择**。也就是说，不从整体最优上加以考虑，它所做出的仅仅是在某种意义上的**局部最优解**。

## 2. 贪心算法的特点
*   **短视（Short-sighted）**：只关注眼前的利益，不考虑长远后果。
*   **高效（Efficient）**：通常只需要一次遍历（$O(n)$），不需要像动态规划那样记录大量中间状态。
*   **无后效性**：某个状态以前的过程不会影响以后的状态，只与当前状态有关。

---

## 3. 贪心算法三部曲
1.  **分解子问题**：将大问题拆解为一个个小步骤。
2.  **制定贪心策略**：在每一步中选择局部最优解。
3.  **证明（或直觉判定）**：局部最优解的堆叠是否能导致全局最优解。

---

## 4. 实战案例：LeetCode 55 (跳跃游戏)

### 题目逻辑
给定一个非负整数数组，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标。

### 贪心策略
*   **不纠结怎么跳**：我们不关心是通过哪几步跳到终点的。
*   **只看势力范围**：我们维护一个变量 `maxReach`，代表当前能达到的最远下标。
*   **核心逻辑**：只要当前位置 `i` 在我们的“势力范围”内，我们就尝试利用当前位置的跳跃能力来**扩张**势力范围。

### 代码片段
```typescript
let maxReach = 0;
for (let i = 0; i < nums.length; i++) {
    if (i > maxReach) return false; // 掉队了，走不到这里
    maxReach = Math.max(maxReach, i + nums[i]); // 扩张领土
    if (maxReach >= nums.length - 1) return true; // 领土覆盖终点
}
```

## 5. 贪心 vs 动态规划 (DP)
| 特性 | 贪心算法 | 动态规划 |
| :--- | :--- | :--- |
| **选择方式** | 仅根据当前信息做选择 | 综合之前的子问题结果做选择 |
| **效率** | 极高 ($O(n)$) | 较高，但需额外空间 ($O(n^2)$ 或 $O(n)$) |
| **最优性** | 不一定能得到全局最优 | 只要状态方程正确，必能得到最优 |

**一句话区别**：贪心是“只顾眼前”，动态规划是“运筹帷幄”。

---

## 6. 适用场景建议
*   当你发现问题的最优解可以通过一系列局部最优的选择达到时。
*   当你在解题时发现“我只需要关注当前最远/最大/最小的上限，而不需要管我是怎么得到它的”时，优先尝试贪心。
