# 桶排序 (Bucket Sort) 学习笔记

## 1. 算法简介
桶排序是一种**非比较型**排序算法，它将待排序的数据分到有限数量的“桶”里，每个桶再分别排序（有可能再使用别的排序算法或是以递归方式继续使用桶排序进行排序）。

## 2. 核心原理
桶排序工作的基本思想是：
1. **分桶**：根据数据的范围，人为设置一个区间的宽度（Width），将大范围切分成若干个连续的子范围。
2. **入桶**：遍历原始数据，根据某种映射函数（如 `Math.floor(value / width)`），将每个数据放入对应的桶中。
3. **桶内排序**：对每个桶内的数据进行单独排序（如果只是为了找范围内的数，如 LC 220，则这一步可以演变为检查相邻桶）。
4. **合并**：按照桶的顺序，将每个桶里的数依次取出，得到最终有序序列。

---

## 3. 算法复杂度
*   **平均时间复杂度**：$O(n + k)$，其中 $n$ 是元素个数，$k$ 是桶的个数。
*   **空间复杂度**：$O(n + k)$，需要额外的空间来存储桶及其内部的元素。

## 4. 实战案例：LeetCode 220 (存在重复元素 III)

这道题是桶排序思想的**变相应用**（主要利用其映射关系，而非排序结果）。

### 题目要求
找出一对 $(i, j)$，满足：
1. $|i - j| \le indexDiff$ (索引距离限制)
2. $|nums[i] - nums[j]| \le valueDiff$ (数值差异限制)

### 为什么用桶排序？
如果使用滑动窗口内找最大/最小值，时间复杂度通常是 $O(n \cdot k)$。通过桶排序，我们可以将查找优化到 **$O(1)$**。

### 桶设计方案
*   **桶宽 (Width)**：设为 `valueDiff + 1`。
*   **映射规则**：`id = Math.floor(num / (valueDiff + 1))`。
*   **判定条件**：
    1. 如果两个数落入**同一个桶**：它们之间的差值绝对不会超过 `valueDiff`。
    2. 如果两个数落入**相邻桶**：它们之间的差值**可能**满足 $\le valueDiff$，需要额外检查。
    3. 如果两个数落入**非相邻桶**：它们之间的差值绝对大于 `valueDiff`。

### 代码逻辑实现
```typescript
const width = valueDiff + 1;
const id = Math.floor(num / width);

// 存在性检查 (O(1))
if (buckets.has(id)) return true;
if (buckets.has(id - 1) && Math.abs(num - buckets.get(id - 1)) <= valueDiff) return true;
if (buckets.has(id + 1) && Math.abs(num - buckets.get(id + 1)) <= valueDiff) return true;

// 窗口控制
buckets.set(id, num);
if (i >= indexDiff) {
    buckets.delete(Math.floor(nums[i - indexDiff] / width));
}
```

---

## 5. 总结与建议
*   **适用场景**：数据分布均匀，且需要处理**范围查找**或**近似相等**的问题。
*   **优势**：在特定条件下能达到线性时间复杂度 $O(n)$。
*   **局限**：如果数据极其不均匀（例如大量数据挤在一个桶里），效率会退化。但在解决 LeetCode 220 这种“范围判定”问题时，桶排序是绝对的利器。
